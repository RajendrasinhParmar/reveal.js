What is ReactJS?

  • Developed by Facebook
  • It is a view library
  • It is not a framework like Angular, Backbone etc.
  • We can not use React to build a fully functional web app without using third-party dependencies
  • Initial release: March 2013; 5 years ago
  • Announced on April 18, 2017 React Fiber is an algorithm to build user interface(React 16)
  • ReactJS is javascript based view library

Why was React developed?

  • Two-way data binding complexity
  • Bad UX due to direct DOM tree manipulation
  • Data volume on a page that changes over time
  • Complexity of Facebook's UI Architecture and so many elements on page
  • One-way data flow makes debugging and developing app easy
  • Asana, Facebook, Airbnb, Netflix, New York Times and many more

React: Pros

  • Easy to understand what a component will render
    • Declarative code → predictable code
    • We don't really need to study js in view file to understand what actually the file does
  • Easy to mix HTML and JS
    • We are doing it already by using template libraries like ejs, Handlebars, Mustache
    • In this case we will use syntax called JSX
  • Uses full power of JS
    Decoupling templates from logic does not rely on the templates’ primitive abstractions, but uses full power of JavaScript in displaying views
  • No complex two-way data flow
    • Use one way reactive data flow
    • Easier to understand than two-way binding
    • Uses less code
  • React is fast!
    • Real DOM is slow
    • JavaScript is fast
    • React uses virtual DOM
    • Virtual DOM objects enables fast batch updates to real DOM
  • React dev tools
    • React Chrome extension makes debugging so much easier
  • Server-side rendering
    • React.renderToString() returns pure HTML

React: Cons

React is nothing but the view

  • No events
  • No XHR
  • No data / models
  • No promises / deferreds
  • No idea how to add all of the above

Architectural annoyances

  • Some time it becomes too complicated to handle all the data changes in deeply nested components if you are using global state for application

Building JSX requires some extra work

  • There are tools available to handle JSX

No support for older browsers

  • React won't work with IE8
  • There some polyfills / shims that help

Why React?

  • Easy to read and understand views
  • Concept of components is the future of web development
  • For fast and real time data update react is a good option
  • Development is faster in react

Virtual DOM

Virtual DOM

  • Efficient re-rendering of DOM
  • Re-render the virtual tree only when state changes
  • Use 2 virtual tree to find the difference
  • Batch update real DOM
  • Observe data changes (setState) and does dirty check to know when to re-render component
  • Doesn't refresh entire DOM
Whenever possible, does not update entire component in real DOM - only computes a patch operation that updates part of the DOM

Hooks

  • Hooks are functions that let you “hook into” React state and lifecycle features from function components
  • Hooks don’t work inside classes
  • they let you use React without classes
  • Hooks are a new addition in React 16.8. They let you use state and other React features without writing a class
  • Hooks are backwards-compatible

Hooks

Basic Hooks

  • State Hook (useState)
  • Effect Hook (useEffect)
  • Context Hook (useContext)

Hooks

Additional Hooks

  • useReducer
  • useCallback
  • useMemo
  • useRef
  • useImperativeHandle
  • useLayoutEffect
  • useDebugValue
  • useReducer is usually preferable to useState when you have complex state logic that involves multiple sub-values or when the next state depends on the previous one.
  • useCallback will return a memoized version of the callback that only changes if one of the dependencies has changed
  • useMemo will only recompute the memoized value when one of the dependencies has changed.
  • useRef is like a “box” that can hold a mutable value in its .current property
  • useDebugValue can be used to display a label for custom hooks in React DevTools.
