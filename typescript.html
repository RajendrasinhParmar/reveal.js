Installation

You will need Node.js and Npm for this demo. Go here if you don't have them installed.
We can install TypeScript using npm with following command. Open terminal(Mac/Linux)/command prompt(Windows). 
npm install -g typescript
Check if typescript is installed properly with following command. 
tsc -v

Compilation

  • Written in .ts files (or .tsx for JSX)
  • Browser can not direct understand typescript
  • Need to be translated to vanilla .js first

Different method for compilation

  • In terminal using command line tool tsc
  • Directly in Visual Studio or some of the other IDEs
  • Using automated task runners such as gulp
The following command takes a TypeScript file named main.ts and translates it into its JavaScript version main.js. If main.js already exists it will be overwritten. 
tsc main.ts
Can also compile multiple files at once 
# Will result in separate .js files: main.js worker.js.
tsc main.ts worker.ts
Can also compile by applying wildcards 
# Compiles all .ts files in the current folder. Does NOT work recursively.
tsc *.ts
Use the --watch option to automatically compile a TypeScript file when changes are made 
# Initializes a watcher process that will keep main.js up to date.
tsc main.ts --watch
More advanced TypeScript users can also create a tsconfig.json file, consisting of various build settings. You can read more about tsconfig.json in the TypeScript docs here

Static Typing

TypeScript supports static typing Can declare the types of variables, and the compiler will make sure that they aren't assigned the wrong types of values If type declarations are omitted, they will be inferred automatically from your code

Most commonly used data types

  • Number - All numeric values are represented by the number type, there aren't separate definitions for integers, floats or others.
  • String - The text type, just like in vanilla JS strings can be surrounded by 'single quotes' or "double quotes".
  • Boolean - true or false, using 0 and 1 will cause a compilation error.
  • Any - A variable with this type can have it's value set to a string, number, or anything else.
  • Arrays - Has two possible syntaxes: 
    my_arr: number[]; or my_arr: Array<number>
  • Void - Used on function that don't return anything.

Example

var name: string = 'Rajendrasinh',     // String 
    age: number = 26,           // Numeric

// Alternatively, you can omit the type declaration:
// var name = 'Rajendrasinh';

// The function expects a string and an integer.
// It doesn't return anything so the type of the function itself is void.

function speak(name: string, age: number): void {
  console.log("Hello " + name + " you are" + age + " old");
}

speak(name, age);
If we try to do something illegal, on compilation tsc will warn us that there is an error in our code. 
// The given type is boolean, the provided value is a string.
var isSwitchOn: boolean = "I haven't checked";
main.ts(1,5): error TS2322: Type 'string' is not assignable to type 'boolean'.
Same is true for passing invalid type of arguments to the function

Interfaces

Interfaces are used to type-check whether an object fits a certain structure When translated to JavaScript, interfaces disappear

Example

// Here we define our Person interface, its properties, and their types.
interface Person {
    name: string;
    age: number;
}

// We tell our function to expect an object that fulfils the Person interface. 
// This way we know that the properties we need will always be available.
function speak(person: Person): void{
  console.log("Hello, " + person.name + " you are " + person.age + " years old.");
}

// We define an object that has all of the properties the Person interface expects.
// Notice that types will be inferred automatically.
var person1 = {
  name: "Rajendrasinh", 
  age: 26
}

speak(person1);
